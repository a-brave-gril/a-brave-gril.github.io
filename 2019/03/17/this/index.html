this指向总结

Veröffentlicht am 2019-03-17 | in 前端笔记-经验小结

前端的this指向一直是一个让人头疼的问题，很容易就会分不清当前this是哪一个，这篇博客就是讲解一个各种情况下面的this指向问题。可能总结的不是很全面，欢迎大家继续补充。

this的指向现在分为两个大方面：普通函数的this指向，以及箭头函数的this指向。

普通函数的this指向

1. 普通的函数不同的写法this的指向也是不一样的。下面是典型的三种例子：
    1. 普通的方法： this永远指向的是最后调用它的对象，也就是看他执行的时候是谁调用的。
    举例说明：
    var obj = {
        name:"ceshi",
        fun:function(){
            console.log(this.name);
        }
    };
    obj.fun() // "ceshi"
    注意：这里有一个坑，就是var fun = obj.fun; fun(); 此时输出的值为：undefined。因为调用fun的是window，所以此时的this指向的是window

    2. 构造函数
        构造函数的this指向与构造函数的而返回值有很大的关系。
        举例说明
        var Person = function(name){
            this.name = name;
        }
        var person = new Person("ceshi");
        person.name // ceshi  此时的this指向的是构造函数的实例。
        构造函数的new关键字，可以改变this的指向。


        var Person = function(name){
            this.name = name;
            return {}
        }
        var person = new Person("ceshi");
        person.name // undefined   此时输出的是“undefined”，因为构造函数返回的是对象。此时this指向的是构造函数的返回值

        注意：当返回的是null的时候结果又不一样了
        var Person = function(name){
            this.name = name;
            return null
        }
        var person = new Person("ceshi");
        person.name // ceshi    此时输出的是“ceshi”,虽然null也代表的是空对象，但是this还是指向构造函数的实例
        总结：在写构造函数的时候最好是没有返回值或者返回this


    3. call、apply以及bind绑定this的函数
        call、apply以及bind的作用都是用来改变this的指向的。
        但是call和apply与bind是有区别的：call和apply定义后会立即执行，但是bind不会立即执行，bind将绑定好的this重新返回一个新函数，在调用的时候执行
        而bind 和 apply主要是在传参数的方式上有区别

2.  箭头函数的this指向
箭头函数的this指向是在定义时的对象，而不是使用时所在的对象。即this不会因为调用对象的改变而改变，当然这是的call、bind以及apply也就没作用了。
但是箭头函数也不是什么情况下都适用的，比如：箭头函数不可以适用new命令，这样写是会报错的
