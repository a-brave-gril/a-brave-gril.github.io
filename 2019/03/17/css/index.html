css原理小结

Veröffentlicht am 2019-03-17 | in 前端笔记-经验小结

最近看了一些css样式的原理，下面是一些学习心得，如果有误欢迎指正。

position

1.      position:static 默认值，无任何效果
2.      position:relative 没有脱离文档流，相对于自己的位置定位。
3.      position:absolute 脱离文档流，是根据最近的不是static的父元素定位的，如果都没有定位，则根据浏览器窗口定位。、
4.      position:flex   相对于浏览器窗口定位，不脱离文档流。

display

1.      display:inline-black  换行符会生成一个空白符，解决方法：父元素font-size：0，但是谷歌不支持，也可以用letter-spacing（控制文字间的水平距离）
2.      display:inline-black 不会像float一样由于高度不一致导致排版失效，所有的inline-black和inline一样会把最高元素的高当做整行元素的高

vertical-align

1.      vertical-align  用来指定行内元素(inline-black 和 inline)或者单元格元素的垂直对齐方式
2.      display:inline-black 不会像float一样由于高度不一致导致排版失效，所有的inline-black和inline一样会把最高元素的高当做整行元素的高。

visibility 和 display的区别

1.  占据空间：visibility还占据空间，但是display不占据空间
2.  回流与重绘：display:none会引起回流和重绘，但是visibility:hidden不会。
3、 株连性：父设置display:none时，子元素无论怎样都不会展示，但是如果父设置了visibility:hidden，子再重新设置一下还是会显示出来的。

注意事项

1.  脱离文档流的有：display:none、绝对定位、浮动
2.  visibility:hidden 隐藏dom,但是在文档流中还占这位置。
