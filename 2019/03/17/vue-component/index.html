vue组件传值以及问题总结

Veröffentlicht am 2019-03-17

介绍

vue组件能够高效的复用代码，并且将公用的功能提取出来也有利于代码的维护。但是在编写vue组件的时候，经常会遇到组件之间的数据传递的问题。下面是我总结的组件之间传值的种类，以及我工作中遇到传值问题的解决方法。

父组件传值给子组件

1、父组件通过 Prop 向子组件传递数据
举例说明；
父组件；<component :post-data="dataList"></component> //注意:post-data一定不是驼峰命名，因为HTML是不区分大小写的
子组件；
Vue.component('blog-post', {
  props: ['postData'],
  template: '<h3>{{ postData }}</h3>'
})   //用来接收父组件传递的数据

子组件传递给父组件

1、子组件通过事件像父组件传值的
举例说明；
父组件；<component @sendMessage="getChildMessage"></component>
子组件；
模板写法：  <div @click="propEvent"></div>
事件语法；  methods{
                propEvent(){
                    $emit('sendMessage','传递的值')
                }
            }

兄弟组件之间相互传值

1、let vueEvent = new Vue();    //先写一个公用的JS，创建一个vue实例。
2、vueEvent.$emit("data",'传递的数据')  //在兄弟组件1上注册一个自定义的事件,用于传送数据
3、vueEvent.$on("data",(data) => {console.log(data)})  //在兄弟组件2上获取兄弟组件1传递的数据
解释：vm.$emit；触发当前实例上的事件，附件参数都会传给监听器回调
      vm.$on: 监听当前实例上的自定义事件，事件可以由vm.$emit触发，回调函数会接受所有传入的参数

组件传值的坑

组件传值的时候会踩到很多坑，下面是我遇到坑的总结，希望能帮助你规避一些错误：
1、子组件是不可以直接更改父组件传递过来的值的，如果直接更改vue会报错，报错信息；
Avoid mutating a prop directly since the value will be overwritten whenever the parent component re-renders. 
Instead, use a data or computed property based on the prop's value
解决办法有两种，需要按照不同的情况来定；
![vue组件传值注意事项](vue-component/vue-component.jpg "vue组件传值注意事项")  
2、由于html是不区分大小写的，所以在传值的时候一定注意不可以用驼峰命名要用”kebab-case“命名
